# Setup Scripts - VRAM Detection and Model Selection

## Overview

All three setup scripts (setup.bat, setup.sh, setup.py) have been modified to:
1. Automatically detect GPU VRAM and system RAM
2. Recommend LLM models based on detected hardware
3. Allow interactive model selection
4. Download selected models via Ollama
5. Update JARVIS configuration with selected models

## Changes Made

### 1. jarvis_config/__init__.py
- Added import for `sys` module
- Added user configuration override mechanism
- Now imports and applies `apply_user_config()` from `jarvis_config/user_config.py` if it exists
- This allows user-selected models to override the default MODELS configuration

### 2. jarvis_config/user_config.py (new file)
- Template configuration file that will be auto-generated by setup scripts
- Contains `apply_user_config()` function to override MODELS for planner, verifier, and reasoner
- Keeps MODELS["czech_gateway"] as OpenEuroLLM-Czech:latest
- Can be manually edited by users to change model preferences

### 3. setup.py (Python cross-platform installer)
**New functions:**
- `detect_hardware()`: Detects GPU VRAM (via nvidia-smi), GPU name, and system RAM
- `recommend_models(vram_gb)`: Returns model recommendations based on VRAM size
- `select_model(hardware_info)`: Interactive menu for model selection with custom option
- `update_user_config(model)`: Generates/updates jarvis_config/user_config.py with selected model

**Modified configuration:**
- Changed from fixed `MODELS` list to `BASE_MODELS` (always downloaded)
- Added `SELECTED_MODEL` variable for user's choice
- Updated from 6 steps to 7 steps in installation flow
- Step 5: Hardware detection and model selection
- Step 6: Downloads base models + selected model
- Updated summary to show configuration

**Model Recommendations by VRAM:**
- < 4GB: qwen2.5:3b-instruct (lightweight)
- 4-6GB: qwen2.5:7b-instruct, llama3.1:8b-instruct-q4_K_M
- 6-9GB: llama3.1:8b-instruct-q4_K_M (RECOMMENDED), llama3.1:8b-instruct-q5_K_M, qwen2.5:7b-instruct, gemma2:9b-instruct-q4_K_M
- > 9GB: llama3.1:8b-instruct-q5_K_M, llama3.1:8b-instruct, mistral-nemo:12b-instruct-q4_K_M, llama3.1:70b-instruct-q4_K_M

### 4. setup.sh (Linux/Mac installer)
**New functions:**
- `detect_hardware()`: Detects VRAM via nvidia-smi, RAM via `free` command
- `recommend_models(vram_gb)`: Same logic as Python version
- `select_model(hardware_info)`: Interactive model selection menu
- `update_user_config(model)`: Creates jarvis_config/user_config.py with selected model

**Modified configuration:**
- Changed from `MODELS` array to `BASE_MODELS` array
- Added `SELECTED_MODEL` variable
- Added `RECOMMENDATIONS` array for storing options
- Updated from 6 steps to 7 steps
- Updated summary to show configuration

**Hardware Detection:**
- GPU VRAM: Uses `nvidia-smi --query-gpu=memory.total --format=csv,noheader,nounits`
- GPU Name: Uses `nvidia-smi --query-gpu=name --format=csv,noheader`
- System RAM: Uses `free -m` command

### 5. setup.bat (Windows installer)
**New functions:**
- `detect_hardware()`: Detects VRAM via nvidia-smi, RAM via wmic
- `select_model(hardware_info)`: Interactive model selection with inline recommendation display
- `update_user_config(model)`: Creates jarvis_config/user_config.py with selected model

**Modified configuration:**
- Changed from `MODELS` variable to `BASE_MODELS` variable
- Added `SELECTED_MODEL` variable
- Updated from 6 steps to 7 steps
- Updated summary to show configuration

**Hardware Detection:**
- GPU VRAM: Uses `nvidia-smi --query-gpu=memory.total --format=csv,noheader,nounits`
- GPU Name: Uses `nvidia-smi --query-gpu=name --format=csv,noheader`
- System RAM: Uses `wmic OS get TotalVisibleMemorySize /value`

## Configuration Update

After selection, the setup script creates/updates `jarvis_config/user_config.py`:

```python
"""
User Configuration Override for JARVIS

This file was auto-generated by the setup script based on your
detected hardware and selected model preferences.

You can also edit this file manually to change model settings.
"""

def apply_user_config():
    """Apply user-selected model configuration."""
    import jarvis_config as _cfg

    # Override with user-selected models
    _cfg.MODELS["planner"] = "llama3.1:8b-instruct-q4_K_M"
    _cfg.MODELS["verifier"] = "llama3.1:8b-instruct-q4_K_M"
    _cfg.MODELS["reasoner"] = "llama3.1:8b-instruct-q4_K_M"

    # Note: MODELS["czech_gateway"] should remain as jobautomation/OpenEuroLLM-Czech:latest
```

## Installation Flow

1. **Check Python** - Verify Python 3.10+ is installed
2. **Create Virtual Environment** - Set up isolated Python environment
3. **Install Dependencies** - Install requirements.txt packages
4. **Check/Install Ollama** - Ensure Ollama service is running
5. **Detect Hardware and Select Model** - **NEW: VRAM detection + model selection**
6. **Pull Required Models** - Download nomic-embed-text, OpenEuroLLM-Czech, and selected model
7. **Create Data Directories** - Set up JARVIS data structure
